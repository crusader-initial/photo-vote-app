# Photo Vote App - 界面设计文档

## 应用概述

这是一个匿名照片投票应用，用户可以上传照片创建投票卡片，其他用户在5秒内选择最喜欢的照片。收集10次投票后显示结果统计。

## 屏幕列表

### 1. Home Screen (首页)
- **主要内容**: 两个大按钮入口 - "我要上传" 和 "我要投票"
- **功能**: 角色选择入口，显示今日剩余投票次数
- **布局**: 垂直居中，两个卡片式按钮

### 2. Create Screen (创建卡片)
- **主要内容**: 照片选择区域（2-4张照片网格）
- **功能**: 从相册选择照片，预览已选照片，删除已选照片
- **布局**: 顶部标题，中间照片网格（2x2），底部"下一步"按钮

### 3. SelfGuess Screen (自我预测)
- **主要内容**: 显示已选照片，让用户选择"我以为会被选中的"
- **功能**: 点击选择一张照片作为预测
- **布局**: 顶部提示文字，照片网格，底部确认按钮

### 4. Waiting Screen (等待结果)
- **主要内容**: 进度指示器，显示当前收集票数/10
- **功能**: 实时更新投票进度，提供分享链接
- **布局**: 居中圆形进度环，下方票数文字

### 5. Result Screen (结果展示)
- **主要内容**: 每张照片的选择率和票数
- **功能**: 显示统计结果，标记用户预测是否正确
- **布局**: 照片卡片列表，每张显示百分比和票数

### 6. Vote Screen (投票页面)
- **主要内容**: 2-3张待选照片，5秒倒计时
- **功能**: 点击选择照片，倒计时结束自动跳过
- **布局**: 顶部倒计时进度条，中间照片网格，底部提示

### 7. VoteResult Screen (单次投票结果)
- **主要内容**: 显示刚选择照片的当前选择率
- **功能**: 短暂展示后自动进入下一轮或返回首页
- **布局**: 居中显示选择的照片和百分比

## 主要用户流程

### 上传者流程
1. Home → 点击"我要上传"
2. Create → 选择2-4张照片 → 点击"下一步"
3. SelfGuess → 选择预测照片 → 点击"确认"
4. Waiting → 等待10票 → 自动跳转
5. Result → 查看结果

### 选择者流程
1. Home → 点击"我要投票"
2. Vote → 5秒内选择照片
3. VoteResult → 查看当前选择率 → 自动进入下一轮
4. 重复直到无更多卡片或达到每日上限

## 颜色方案

| 颜色 | 用途 | 值 |
|------|------|-----|
| Primary | 主要按钮、强调色 | #6366F1 (靛蓝色) |
| Background | 页面背景 | #FFFFFF / #151718 |
| Surface | 卡片背景 | #F5F5F5 / #1E2022 |
| Foreground | 主要文字 | #11181C / #ECEDEE |
| Muted | 次要文字 | #687076 / #9BA1A6 |
| Success | 正确预测 | #22C55E |
| Warning | 倒计时警告 | #F59E0B |
| Error | 错误/超时 | #EF4444 |

## 组件设计

### PhotoCard 组件
- 圆角图片容器
- 可选中状态（边框高亮）
- 显示选择率标签

### CountdownBar 组件
- 顶部固定进度条
- 从绿到红渐变
- 5秒动画

### ActionButton 组件
- 大尺寸触摸区域
- 按压反馈动画
- 禁用状态样式

## 数据存储

### 本地存储 (AsyncStorage)
- `deviceId`: 设备唯一标识
- `dailyVoteCount`: 今日投票次数
- `lastVoteDate`: 最后投票日期

### 服务端存储 (PostgreSQL)
- Cards 表: 投票卡片
- Photos 表: 照片信息
- Votes 表: 投票记录
